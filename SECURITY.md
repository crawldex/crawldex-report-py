# Security Policy

This public repository is a generated mirror. The private CrawlDex monorepo remains canonical.

Do not submit real credentials, private customer data, screenshots, cookies, browser traces, downloaded files, or sensitive evidence artifacts in issues, pull requests, examples, or tests.

Report vulnerabilities through GitHub issues only when the report can be written without secrets or sensitive data. For sensitive security reports, contact the CrawlDex maintainer/operator privately and reference this repository: `crawldex-report-py`.

Out of scope: attacks against third-party websites used as public read-only examples, social engineering, denial-of-service against local developer machines, and CAPTCHA or anti-bot bypass requests.
